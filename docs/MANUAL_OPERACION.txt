2. Modos de operacion

DETENER:
sudo systemctl disable openclaw-gateway
sudo systemctl stop openclaw-gateway

ARRANQUE LUEGO DE DETENER:
sudo systemctl start openclaw-gateway
systemctl status openclaw-gateway

AUTO-ARRANQUE = NO se necesita arranque manual, esto duplica los procesos y da problema.
sudo systemctl enable openclaw-gateway
systemctl is-enabled openclaw-gateway


LOGS EN TIEMPO REAL:
journalctl -u openclaw-gateway -f
ss -tulpn | grep 4000


3. PARAR EL GATEWAY (no usar kill)
openclaw gateway stop

Verificar:
pgrep -af openclaw
ss -tulpn | grep 4000

Activar systemd
sudo systemctl enable openclaw-gateway
sudo systemctl reset-failed openclaw-gateway
sudo systemctl start openclaw-gateway

Verificar systemd OK:
sudo systemctl status openclaw-gateway --no-pager
pgrep -a -f openclaw-gateway
ss -tulpn | grep 4000

2.GUIA DE SOLUCION DE PROBLEMAS DE TELEGRAM
Cuando un empleado usa una cuenta de Telegram que ya registro en otro telefono le dira que no lo tiene registrado.

Eso te sacará los números que están llegando al webhook.

sudo journalctl -u talinda-bot-backend --since "24 hours ago" \
| grep "Webhook received" \
| grep -oE '"from":"[0-9]+"' \
| cut -d':' -f2 \
| tr -d '"'

Para contarlos y ver repetidos

sudo journalctl -u talinda-bot-backend --since "24 hours ago" \
| grep "Webhook received" \
| grep -oE '"from":"[0-9]+"' \
| cut -d':' -f2 \
| tr -d '"' \
| sort | uniq -c | sort -nr


Resultado:
     70 8684820190
     43 2098596972
     34 1008343934
     27 5825850746
     23 1399905876
     10 8759559994

Si quieres ver solo números de intentos no registrados
Como no siempre ves el mensaje de respuesta en los logs, una forma práctica es:
1. Extraer números que escribieron

sudo journalctl -u talinda-bot-backend --since "24 hours ago" \
| grep "Webhook received" \
| grep -oE '"from":"[0-9]+"' \
| cut -d':' -f2 \
| tr -d '"' \
| sort -u

 Buscar exactamente alrededor de esa hora

Si 13:38 es hora del servidor
sudo journalctl -u talinda-bot-backend --since "2026-04-09 13:35:00" --until "2026-04-09 13:41:00"
Filtrar solo el webhook
sudo journalctl -u talinda-bot-backend   --since "2026-04-09 10:35:00"   --until "2026-04-09 12:41:00" | grep "Webhook received"
